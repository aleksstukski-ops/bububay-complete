Du arbeitest am Projekt BubuBay.

BubuBay ist ein seller operating system fuer den Eigengebrauch. Ziel ist eine zentrale Plattform, auf der Produkte einmal erfasst oder in grossen Mengen importiert werden und dann plattformgerecht auf mehreren Verkaufskanaelen verwaltet, bearbeitet, synchronisiert und teilweise automatisiert werden.

Produktkern:
- ein Product Master
- mehrere Channel Adapter
- Bulk Import
- Listing Composer
- Orders und Notifications zentral
- Messaging je nach offizieller Kanalfaehigkeit
- Highlights Engine fuer Trend-Discovery und schnelle Reaktion

Arbeitsmodus:
- aktuell nur Eigengebrauch
- keine externen Kunden
- API first
- Browser Automation nur wo noetig
- keine Vermischung mit anderen Projekten oder Domains
- keine unbelegten Behauptungen ueber Kanalfaehigkeiten

Pflicht vor jeder Aufgabe:
1. README.md lesen
2. docs/masterplan.md lesen
3. docs/architecture.md lesen
4. neueste Datei in docs/progress lesen
5. relevante docs/integrations lesen

Pflicht nach jeder Aufgabe:
1. docs/progress aktualisieren
2. naechste Schritte dokumentieren
3. Risiken dokumentieren
4. Docs und Code synchron halten

Arbeitsstil:
- direkt
- realistisch
- kleinste robuste Loesung zuerst
- nichts als erledigt markieren, was nicht verifiziert ist
- Unsicherheit klar benennen
- API-Faehigkeit, Messaging-Faehigkeit, Webhooks, Limits und Partnerzugang pro Kanal getrennt pruefen

Zielbild:
BubuBay soll mindestens die praktischen Kernfaehigkeiten moderner Seller-Tools erreichen und dann darueber hinausgehen, besonders bei:
- Omnichannel Seller Workspace
- Bulk Workflows
- Trend Discovery
- Product-to-Listing Automation
- zentraler Steuerung vieler Kanaele
